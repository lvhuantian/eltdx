# v1.2.0

`v1.2.0` 新增时间安全的短线指标 Helper。它组合 `0x06b9` 下载的 TDX `zhb.zip` 统计资源、实时行情快照、代码表和基准指数日 K，一次返回固定 21 个短线字段，不引入数据库、磁盘缓存或新的二进制命令。

## 调用入口

```python
from eltdx import TdxClient

with TdxClient(timeout=3) as client:
    table = client.helpers.shortline_indicators(
        ["sz000001", "sh600000"],
    )

for row in table.rows:
    print(
        row.full_code,
        row.free_float_market_value,
        row.auction_prev_volume_ratio,
        row.seal_to_float_ratio,
        row.limit_board_text,
    )
```

主要入口是 `client.helpers.shortline_indicators()`；兼容入口 `client.helpers.get_shortline_indicators()` 保持相同参数和返回值。结果模型为 `ShortlineIndicatorTable` 和 `ShortlineIndicator`。

## 21 个指标

统计资源直接提供 12 个字段：

- 近 60 日 Beta、滚动市盈率、流通股本Z。
- 昨成交额、昨封单额、前封单额、昨开盘成交量、昨开盘金额。
- 几天几板统计天数、统计期涨停次数、文件连续涨停天数、文件年内涨停天数。

结合实时行情计算 9 个字段：

- 流通市值Z、开盘换手Z、开盘昨比、竞价昨比。
- 开盘昨封比、封流比、封昨比（昨封比）。
- 几天几板、当前连板高度。

完整字段名、单位和计算公式见[短线指标文档](https://electkismet.github.io/eltdx/helpers/%E7%9F%AD%E7%BA%BF%E6%8C%87%E6%A0%87/)。题材强度、题材排名、题材最高板等概念或行业聚合结果不属于这 21 个字段。

## 时间安全

- 目标交易日来自 TDX 握手，不使用本机日期猜测。
- 上一交易日来自上证指数实际日 K，不用简单的自然日减一天。
- 交易日 09:25 前实时竞价尚未完成，抛出 `ShortlineIndicatorsNotReadyError`。
- `tdxstat.cfg` 和 `tdxstat2.cfg` 的主导统计日期必须一致，且各自至少 95% 的有效行属于该日期。
- 统计日期等于目标交易日时使用文件的 `prev_*` 列；等于上一交易日时，将文件当日列映射为目标日昨日值。
- 文件日期更旧时抛出 `TdxStatsDateError`；少量个股行日期异常时，相关历史分母返回 `None`，不会跨日猜算。

## 缓存边界

已验证的统计资源只在当前客户端内存中缓存。跨到新交易日且旧资源不再可用时会自动重新下载；`refresh_stats=True` 可强制刷新，`client.clear_cache()` 可主动清空。eltdx 不会因此建立数据库或磁盘缓存。

## 兼容性

- 原有行情、K 线、分时、成交、竞价、公司数据、F10 和 Helpers 调用不变。
- 新能力属于 Helper，不改变 `0x06b9`、`0x054c`、`0x0547`、`0x044d` 或 `0x052d` 的底层协议接口。
- `v1.1.0` 引入的单线程非阻塞 `ConnectionActor`、连接池语义和 `pool_size` 默认值保持不变。

## 验证

- 完整测试覆盖 Python 3.10、3.11、3.12 和 3.13，以及 Windows Actor 专项矩阵。
- 短线指标测试覆盖同日、上一交易日、过期文件、CFG 日期冲突、低日期覆盖率、09:25 前未就绪、缓存和强制刷新。
- MkDocs strict、sdist/wheel 构建和 Twine 检查均通过。
