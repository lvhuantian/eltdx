# v1.1.0

`v1.1.0` 重构了 7709 行情传输层，重点解决断线重连后旧 reader 线程可能长期阻塞并持续累积的问题。业务行情、7615 F10 和 Helpers 的调用方式保持不变。

## 连接架构

- 每个连接槽位由一个长期存在的单线程非阻塞 `ConnectionActor` 独占。
- Actor 同时负责 socket、selector、收发、超时和心跳，不再为每次重连创建新的 reader 线程。
- TCP 重连只替换 socket generation，不替换 Actor 线程；旧 generation 的事件不能完成新请求。
- 响应同时校验 runtime epoch、TCP generation、socket identity、lease、`msg_id` 和 `msg_type`。
- Actor fatal 或关闭超时会进入 fail-closed 状态，不会在后台悄悄创建替代线程。

## 连接池调度

- 普通请求改为全池 FIFO admission，通过 exact-once lease 分配给首个空闲 slot。
- `pin()` 返回真正独占单个 slot 的 proxy，退出 context 后只归还 lease，不会关闭共享连接池。
- `pool_size=N` 表示 N 个 Actor、N 条 TCP 连接以及最多 N 个同时在途的网络请求，不代表 N 个行情服务器。
- `TdxClient`、`TdxClient.from_hosts()`、`PooledSocketTransport` 和 `eltdx-smoke` 的默认 `pool_size` 统一为 `1`。
- `pool_size` 必须是正整数；`0`、负数、float、字符串、bool 和 `None` 会直接抛出 `ValueError`，不再被静默截断或改写。

## 有界资源与错误报告

- `timeout` 对数字 IP 和已缓存 endpoint 覆盖排队、连接、握手、发送、响应和一次 retry。
- push queue 改为有界 buffer；溢出时丢弃最旧帧，并通过 `PushOverflowError` 明确报告数据 gap。
- 新增 `max_pending_requests`、`push_queue_size` 和 `push_queue_bytes` 配置。
- 新增 `PoolBusyError`、`PushOverflowError` 和 `TransportCloseTimeoutError`。
- 自定义 hostname 的首次标准库 DNS 解析在 Actor 外执行，不占用 pool slot；该解析仍无法提供严格取消保证。

## 兼容性

- `TdxClient` 的行情、K 线、分时、成交、竞价、公司数据、Helpers 和 7615 F10 业务入口保持兼容。
- 正常使用正整数 `pool_size` 的代码无需修改。
- 依赖字符串、float、bool 或非正数 `pool_size` 自动转换的代码，需要改为显式正整数。
- 为保留 Actor socket 独占、FIFO 和 generation 隔离，正式 A/B 性能验收记录的吞吐约低于 legacy transport 5%～6%；该差异作为已接受的架构取舍保留。

## 验证

- 完整测试覆盖 Python 3.10、3.11、3.12 和 3.13，以及 Windows Actor 专项矩阵。
- 压力验证覆盖 10,000 次 generation 切换和 100,000 个请求，未发现串代、重复或丢失，关闭后受管资源归零。
- 本地完整测试、MkDocs strict、sdist/wheel 构建和 Twine 检查均通过。
