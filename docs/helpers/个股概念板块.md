# 个股概念板块

## 作用

查询某只股票关联的全部概念 / 题材，并尽量合并关联度、入选日期、入选原因和详情记录 ID。

| 项目 | 内容 |
| --- | --- |
| 调用方法 | `client.helpers.stock_topics(code)` |
| 返回模型 | `StockTopics` |
| 底层能力 | [`F10-股票基础信息`](../methods/F10-股票基础信息.md)、[`F10-热点题材`](../methods/F10-热点题材.md) |

## 示例

```python
from eltdx import TdxClient

client = TdxClient(timeout=3)
topics = client.helpers.stock_topics("000034")

for item in topics.topics:
    print(item.topic_id, item.topic_name, item.relation_level, item.reason)
```

## 返回字段

`StockTopics`

| 字段 | 含义 |
| --- | --- |
| `code` | 完整股票代码 |
| `topics` | `StockTopic` 列表 |
| `count` | 题材数量 |

`StockTopic`

| 字段 | 含义 |
| --- | --- |
| `topic_id` | 题材 ID，可继续查题材内成分股 |
| `topic_name` | 题材名称 |
| `relation_level` | 关联度数值 |
| `selected_date` | 入选日期 |
| `topic_date` | 题材日期 |
| `reason` | 入选原因 / 题材说明 |
| `detail_id` | 详情记录 ID |
| `category_raw` | 服务端类别标记 |
| `source` | 合并来源，常见 `topic_ids`、`hot_topics` 或两者都有 |
| `raw` | 合并前的 F10 原始行 |
