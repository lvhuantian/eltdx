# 7709-0925竞价成交快照

## 作用

从历史成交明细 `0x0fc6` 里扫描指定日期 09:25 的竞价成交快照，返回价格、成交量、成交额和方向状态。

| 项目 | 内容 |
| --- | --- |
| 调用方法 | `client.get_auction_0925(code, date)` |
| 数据来源 | [`0x0fc6`](../COMMANDS_7709.md#cmd-0x0fc6) |
| 返回模型 | `Auction0925Result` |

这是 SDK 本地工具方法，数据来自历史成交明细 `0x0fc6`。它会分页拉历史成交明细，然后找 `time_minutes == 09:25` 的记录。

## 示例

```python
from eltdx import TdxClient

with TdxClient(timeout=3) as client:
    result = client.get_auction_0925("sz000001", "2026-05-20")
    if result.has_auction_0925:
        print(result.price, result.volume, result.amount)
```

## 参数

| 参数 | 含义 |
| --- | --- |
| `code` | 证券代码 |
| `date` | 交易日 |
| `page_size` | 扫描历史成交明细时每页条数 |
| `max_pages` | 最多扫描页数 |

## 解析字段

| `Auction0925Result` 字段 | 含义 |
| --- | --- |
| `code` | 完整代码 |
| `trading_date` | 交易日 |
| `has_auction_0925` | 是否找到 09:25 成交 |
| `price` / `price_milli` | 竞价成交价 |
| `volume` | 成交量，单位手 |
| `amount` | 成交额 |
| `status` / `side` | 原始状态 / 方向 |
| `pages_used` | 扫描了几页历史成交明细 |
| `source_mode` | 数据来源说明 |

## 使用建议

如果要看竞价阶段过程，用 [7709-集合竞价明细.md](7709-集合竞价明细.md)。如果只要某天 09:25 最终成交，这个方法更直接。
