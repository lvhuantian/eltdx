# 7709-服务器文件读取

## 作用

通过 `0x06b9` 按路径、偏移和长度读取一个通达信服务器文件块。接口返回单块模型，其中 `content` 是原始 bytes；不解析文件格式。

| 项目 | 内容 |
| --- | --- |
| 调用方法 | `client.resources.read(path, offset=0, size=30000)` / `client.read_server_file(...)` |
| 底层接口 | [`0x06b9`](../COMMANDS_7709.md#cmd-0x06b9) |
| 返回模型 | `FileContentChunk` |

## 示例

```python
from eltdx import TdxClient

with TdxClient(timeout=3) as client:
    chunk = client.read_server_file("zhb.zip", offset=0, size=30000)
    print(chunk.chunk_len, chunk.is_last)
```

## 参数

| 参数 | 含义 |
| --- | --- |
| `path` | 服务器文件路径；必须是 ASCII，最长 300 字节，反斜杠会转成 `/` |
| `offset` | 文件读取偏移，`uint32`，默认 `0` |
| `size` | 请求字节数，范围 `1..60000`，默认 `30000`；`60000` 是服务端单次响应上限 |

## 返回字段

| `FileContentChunk` 字段 | 含义 |
| --- | --- |
| `path` | 规范化后的服务器路径 |
| `offset` | 本次读取偏移 |
| `request_size` | 本次请求字节数 |
| `chunk_len` | 服务端实际返回字节数 |
| `content` | 文件块原始 bytes |
| `raw_payload` | 包含长度头的原始响应 payload |
| `is_last` | `chunk_len < request_size` 时为 `True` |

该接口只读取单个文件块，不包含整文件循环下载、ZIP 解包、缓存或文件内容解析。
