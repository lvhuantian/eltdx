# 7709-旧版批量行情

## 作用

按显式代码列表调用 `0x053e` 旧版批量行情接口，返回行情、五档盘口和协议状态原始字段。

| 项目 | 内容 |
| --- | --- |
| 主要调用 | `client.quotes.legacy(codes)` |
| 自动拆批 | `client.get_legacy_quotes(codes)` |
| 底层接口 | [`0x053e`](../COMMANDS_7709.md#cmd-0x053e) |
| 返回模型 | `list[LegacyQuote]` |

## 与 `0x054c`、`0x0547` 的区别

`0x053e` 的核心定位是**无游标的旧版完整快照**，每次请求都返回当时的基础行情、完整五档和旧版状态字段。

- 它不是 `0x054c`：`0x054c` 是当前基础快照，实盘稳定确认一档盘口，不提供 `0x053e` 的交易状态和旧版尾部指标。
- 它不是 `0x0547`：`0x0547` 按代码和游标刷新行情，并可配合推送队列；`0x053e` 没有游标或连续刷新语义。
- 只需要常规当前行情时优先使用 `client.get_quote()`；只有兼容旧协议或需要旧版原始状态字段时才使用 `0x053e`。

## 示例

```python
from eltdx import TdxClient

with TdxClient(timeout=3) as client:
    quotes = client.get_legacy_quotes(["sz000001", "sh600000"])
    print(quotes[0].last_price, quotes[0].buy_levels)
```

`client.quotes.legacy()` 直接发送一次请求；`client.get_legacy_quotes()` 会按客户端 `batch_size` 自动拆批，默认最多 80 个代码一批。

## 参数

| 参数 | 含义 |
| --- | --- |
| `codes` | 单个代码或代码列表，支持 `sz000001`、`sh600000`、`bj920856` 等完整代码 |

## 解析字段

| `LegacyQuote` 字段 | 含义 |
| --- | --- |
| `exchange` / `market_id` | 市场前缀 / 市场编号 |
| `code` / `full_code` | 六位代码 / 完整代码 |
| `active1` | 记录头部状态原始值 |
| `last_price` / `pre_close_price` | 最新价 / 昨收价 |
| `change` / `change_pct` | 涨跌额 / 涨跌幅；昨收为 0 时涨跌幅为 `None` |
| `open_price` / `high_price` / `low_price` | 今开 / 最高 / 最低 |
| `server_time_raw` | 服务端行情时间原始值 |
| `unknown_after_time_raw` | 行情时间后的未命名协议原始值 |
| `total_hand` / `current_hand` | 总成交量 / 现手，单位手 |
| `amount` / `amount_raw` | 成交额解析值 / 协议原始值 |
| `inside_dish` / `outer_disc` | 内盘 / 外盘 |
| `unknown_after_outer_raw` | 外盘后的未命名协议原始值 |
| `open_amount_raw` / `open_amount_yuan` | 开盘金额原始值 / 元值 |
| `buy_levels` / `sell_levels` | 买一到买五 / 卖一到卖五 |
| `sum_buy_vol` / `sum_sell_vol` | 五档买量合计 / 五档卖量合计 |
| `trading_status_raw` / `trading_status_hex` | 交易状态原始值 / 十六进制文本 |
| `tail_metrics_raw` | 四个尾部指标原始值 |
| `rise_speed_raw` / `active2` | 可选旧版尾部字段；短尾响应中为 `None` |
| `record_hex` | 单条记录原始十六进制 |

`buy_levels` / `sell_levels` 中的单档为 `QuoteLevel`，包含 `price`、`volume` 和 `price_delta_raw`。

这个接口只返回底层行情数据，不执行股票筛选或状态分类。
